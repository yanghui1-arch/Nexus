DDiscord 入口

Bot 在私有 server 中运行

Bot 不像真人一样加好友。Nexus 使用 bot token 调 Discord API，并用你的 user ID 创建或复用 DM channel。

轮询你的 DM 命令

后台 poller 读取你发给 bot 的新消息，只处理配置的 Discord user ID，避免其他人触发 review。

scan review owner/repo#123 review https://github.com/owner/repo/pull/123 status owner/repo#123

命令结果回到 DM

手动命令会先回复 task queued/status。Assistant 完成后只在重要合并、拿不准，或明确 Discord 手动命令需要结果时 DM。

AAssistant Review Task

入口只负责派发 task

来源 A：Discord 的 review 或 scan 命令。来源 B：定时扫描 active Assistant agent workspace 绑定 repo。两者都会创建 TaskCategory.review 的 celery task。

celery 创建 Assistant agent

Assistant 是普通 Nexus agent class，继承现有 Agent，使用自己的 workspace、GitHub tools、sandbox tools 和 Discord tools。

读取 PR、checkout 并测试

Assistant 使用 GitHub tools 读取 PR metadata、diff、reviews、comments、checks，然后在 sandbox 中 checkout PR head，并运行配置的测试命令。

Assistant 提交正式 review

实质问题 REQUEST_CHANGES，通过时 APPROVE，CI pending、测试缺失、merge conflict 等只 COMMENT。

完成 review task

task 结束后状态进入 closed，结果写入普通 task result；不会进入 coding task 的 work-item/merge review 状态机。

GGitHub Gate 与输出
自动合并必须全部满足
1PR open、非 draft、head SHA 未变
2GitHub mergeable clean，无 merge conflict
3CI/checks/statuses 全部成功，不能缺失
4配置的本地测试命令通过
5Assistant 本轮正式 review 是 APPROVE
6没有有效的 human changes requested

Gate 通过

Assistant 使用 merge_pr tool，并传入 expected sha=head_sha；GitHub 会拒绝 head 已变化的合并。

Gate 阻塞

不合并。已知阻塞点写入 GitHub review/comment；只有拿不准时才 DM。

Discord DM 策略
静普通合并：不 DM
静Merge conflict：不 DM，只在 PR 下留言
DM重要合并：DB/migration、auth/security、依赖、部署、公共 API、计费、大 diff
DM拿不准：verdict uncertain、策略/权限阻塞且需要你判断

持久化

task 表新增 review category，并用 external_pull_request_url 绑定 PR；secretary_state 保存 Discord 命令游标和 scan head 去重。

安全边界

自动合并发生在 Assistant 的 review task 内，但必须通过工具和 GitHub expected SHA guard；入口 poller/Discord command 不直接 review 或 merge。

默认关闭

只有 NEXUS_SECRETARY_ENABLED=true、Assistant GitHub token 已配置，且存在 active Assistant workspace repo 时，poller 才会扫描并派发 review task。

C运行配置摘要
配置项 作用
NEXUS_SECRETARY_ENABLED 启用或关闭 secretary poller。
NEXUS_ASSISTANT_GITHUB_TOKEN assistant agent 的 GitHub token，用于读取 PR、提交 review、按 gate 自动 merge。
Assistant workspace repo 后台扫描 active Assistant agent instance 对应 workspace 绑定的 github_repo，并派发 review task。
NEXUS_SECRETARY_DISCORD_BOT_TOKEN Discord bot API token，用来创建 DM、发消息和读取你的命令。
NEXUS_SECRETARY_DISCORD_USER_ID 只接受这个 Discord 用户的命令，并只向这个用户发低噪音 DM。
NEXUS_SECRETARY_TEST_COMMANDS_JSON repo 到测试命令列表的 JSON 映射，例如 {"owner/repo":["pytest"],"*":["pytest"]}。
